## 과거의 자바 정의
지금까지 자바라는 언어를 단순히 익히기에 급급했던 것 같다.   
그래서 동시성이라는 개념에 부딪혔을 때 이해하지 못하고 어떻게 사용하는가라는 잘못된 생각을 하게 되었다. 
처음으로 돌아가 자바의 JVM에 대해 정리를 하고자 한다.

## JVM이란?
JVM(Java Virtual Machine)은 자바 바이트 코드를 실행해주는 가상 머신이고 플랫폼에 독립적이다.    
익숙하지만 하나씩 뜯어보면 모르는 용어들이 많다.

### 바이트 코드를 실행?
예전부터 자바를 공부를 하면서 자바 코드를 바이트 코드로 컴파일 된다라는 것은 이미 알고 있었지만
  
바이트 코드를 실행한다?  
  
이 전에 바이트 코드와 바이너리 코드의 차이를 알아야 한다.  
  
바이너리 코드는 CPU가 인식할 수 있는 0과 1로 이루어져 있는 이진코드
바이트 코드는 가상머신이 이해할수 있는 코드  

그럼 바이트 코드를 실행해준다함은 JVM이 이해할수 있는 코드,  
즉 바이트 코드를 JVM이 해석한다는 것이다.  
더 정확히는 JVM의 실행엔진은 바이트 코드를 CPU가 실행할 수 있는 바이너리 코드로 컴파일한다.(물론 컴파일 방식에는 또 나뉘지만)


### 플랫폼에 독립적?
CPU에 따라 기계어의 설계가 다르고 OS마다 인식하는 것이 다르기 때문에
어떤 기계어를 여러 환경에서 실행을 할 수 없다.  
하지만 자바는 자바 코드를 JVM가 이해할 수 있는 바이트 코드로 컴파일하고 각 환경에 JVM만 있으면  
그 OS가 해석할 수 있는 바이너리 코드로 컴파일하기 때문에 플랫폼에 독립적이게 된다.

## 자바 실행 과정
자바 실행 과정을 간단하게 정리하자면  
1. 자바코드(.java)는 자바 컴파일러(javac.exe)에 의해서 바이트 코드(.class)로 변환이 된다.
2. 컴파일된 바이트 코드는 클래스 로더에 의하여 JVM에 적재가 된다.
3. 그 적재된 코드는 execution engine에 의하여 바이트 코드를 해석되어 실행하게 된다.

### 인터프리터, JIT 컴파일러 방식
아까전에 바이트 코드를 JVM이 해석을 한다고 했었는데 사실  
해석하는 방식에는 인터프리터 방식, JIT 컴파일러 방식이 있다.  
바이트 코드를 실행할 때
한번 실행될 코드는 인터프리터로 실행을 하고 주기적으로 사용될 코드들은 체킹을 하고 
일정 횟수가 넘어가면 JIT로 컴파일을 한다.
